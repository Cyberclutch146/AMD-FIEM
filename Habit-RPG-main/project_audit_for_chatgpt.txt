# Project Audit: Habit Tracker RPG (Aether HUD)

This document provides a detailed technical and conceptual overview of the project to help an AI assistant (like ChatGPT) understand the codebase, its architecture, and its features.

## 1. Project Concept
The **Habit Tracker RPG** is a gamified productivity application. It transforms real-life habit completion into progress within a 2D RPG environment. Users track habits (e.g., workouts, diet, steps) to earn XP, gold, and deal damage to a series of bosses.

## 2. Technical Stack
- **Frontend**: React 18 (Vite-based), TypeScript.
- **Styling**: 
  - **Tailwind CSS**: Core layout and responsive design.
  - **Framer Motion**: High-fidelity animations, transitions, and "game juice" (e.g., floating XP, level-up effects).
  - **Lucide React & Material Symbols**: Iconography.
- **State Management**: **Zustand**. 
  - `useHabitStore`: Manages habits and completion logs.
  - `useUserStore`: Manages user/character profile and stats.
  - `useJuiceStore`: Handles temporary visual effects (animations).
- **Backend (BaaS)**: **Firebase**.
  - **Firestore**: Real-time NoSQL database.
  - **Authentication**: User login and profile management.
- **Validation**: **Zod** (used for schema definition and runtime validation of Firestore data).
- **Utilities**: `date-fns` for date manipulation and streak calculations.

## 3. Project Structure
- `/src/components`: UI components (layout, animations, shared widgets).
- `/src/pages`: Application screens.
  - `Dashboard.tsx`: Habit list and check-in.
  - `Boss.tsx`: Visualized boss encounters and progression gauntlet.
  - `Stats.tsx`: User profile, rank visualization, and performance charts.
  - `Vault.tsx`: Inventory management and theme customization.
  - `Login.tsx`: Firebase auth integration.
  - `Settings.tsx`: App configuration.
- `/src/lib`: Core logic and services.
  - `gameEngine.ts`: Centralizes all RPG math (XP curves, combat multipliers, loot generation).
  - `db.ts`: Abstraction layer for Firestore with Zod-typed collections.
  - `firebase.ts`: Firebase initialization.
  - `services/`: Specific business logic for `users` and `habits`.
- `/src/store`: Zustand store definitions.

## 4. Key Game Mechanics
- **Progression System**:
  - **XP & Leveling**: Uses a power curve `BASE_XP * level^1.5`. Leveling up grants skill points.
  - **Ranks**: 12 ranks from `UNRANKED` to `GOD`, with subdivisions (e.g., Gold III, II, I).
- **Combat System**:
  - **Bosses**: A roster of 7 bosses (e.g., Sloth Demon, Void Titan) with increasing HP.
  - **Damage**: Calculated based on the habit's XP reward, combo multipliers, and class specialization.
  - **Weaknesses**: Bosses are vulnerable to specific habit types (e.g., Workout, Diet), which doubles damage.
  - **Crits**: Equipped weapons provide a 15% chance for critical hits (2x damage).
- **Economy & Items**:
  - **Gold**: Earned via habits; intended for unlocking themes and items.
  - **Loot**: Dropped upon boss defeat. Items have rarities (Common to Legendary) and stat bonuses (e.g., "hp+10%").
- **Retention Mechanics**:
  - **Streaks**: Daily completion builds streaks.
  - **Streak Shields**: Consumable items that protect a streak if a single day is missed.
  - **Classes**: Warrior, Mage, Rogue – each provides damage bonuses to specific habit types.

## 5. Data Architecture (Firestore)
- **`users/{userId}`**: Stores profile, level, gold, streak, class, and inventory.
- **`users/{userId}/habits/{habitId}`**: Definitions for habits (title, difficulty, type).
- **`users/{userId}/logs/{logId}`**: Immutable records of habit completions, including damage dealt and rewards earned. Uses idempotency keys (`userId_habitId_date`) to prevent duplicate check-ins.

## 6. Design Aesthetic
The project uses a premium, "Aether HUD" aesthetic:
- **Dark Mode**: Deep backgrounds with high-contrast primary colors.
- **Glassmorphism**: Backdrop blurs and translucent containers.
- **Kinetic UI**: Frequent use of `AuroraBackground`, `GlitchText`, and `StarBorder` animations to create a "living" interface.

## 7. Current State & Known Patterns
- The project is fully typed and uses strict TypeScript rules.
- Optimistic UI updates are used for habit completion to ensure a "snappy" feel, with background syncing to Firebase.
- The `gameEngine.ts` is the single source of truth for all mathematical calculations.
